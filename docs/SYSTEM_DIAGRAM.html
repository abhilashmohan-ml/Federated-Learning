System Overview

Top-level view of the five manufacturing sites, the central FL server, and the shared library. Each site runs an isolated Docker container. Only gradient updates (ΔW) travel between containers — raw filtration data never leaves the site.

  • Shared Library — physics models, schemas, crypto: used by both server and all clients
  • FL Server — aggregates model updates, manages rounds, issues JWTs
  • Site Clients (×5) — load data locally, train, add DP noise, upload ΔW

Federated Learning Protocol Flow

Sequence of calls across one complete FL round. Each round produces a new version of the global model. The sequence repeats FL_ROUNDS times (default 50).

Docker Network Isolation

Network topology in the Docker Compose deployment. Each site is isolated on its own bridge network. Sites cannot communicate with each other or with the database directly — they can only reach the server through their shared network segment.

  • db_net: server ↔ PostgreSQL only
  • site_N_net: server ↔ site_N only (one per site)
  • Sites cannot reach each other (mirrors production topology)
